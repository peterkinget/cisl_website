Switcap3 User Guide.

Installing Switcap3:

Download the following binaries:
sw3

SUN version
HP version

 This is the binary that contains the noise analysis. For other types of analyses use Switcap2.
*Note*: The binary will only work on machines whose hostid was given to us.
psw3

SUN version
HP version

 This is a post processor that is used to format the data produced by sw3.

Noise analysis theory

The noise analysis in Switcap3 is based on the theory described in Cahpter 5 of the folowing reference:
 
I. Yusim, Simulation of Switched-Capacitor and Switched-Current Networks Under Instant Settling Approximation, PhD Thesis, Columbia University, New York, 1999.

Netlist Syntax

    The syntax is backward compatible with Switcap2. See the Switcap2 manual (postscript version). The new features are:
  • Noisy switch
  • Noisy VCVS
  • Ideal op-amp (noisy and noiseless)
  • Noise analysis block
See the file filter.scn for an example of the new Switcap3 syntax.
 

Running Switcap3

To run Switcap3 use the command:
sw3 < netlist.scn
Switcap3 will produce the file "sw3.noise". This file contains simulation results in binary form. (In the future release we will allow the user to select the name for the output file.) To extract the data from "sw3.noise" use the post processor psw3.

The output file "sw3.noise" will contain the following data.

  • Noise PSD for each selected output with respect to each noisy element -- white and 1/f contribution separately.

    •  
  • The total noise PSD for each selected output due to all noisy elements -- white and 1/f contributions combined.

Using the post processor (psw3)

To see the names of the wave forms that are in the output file "sw3.noise", use the command:
psw3 -l < sw3.noise
To print the data in a table format use the command:
psw3 waveform_names < sw3.noise
Note, multiple wave form names can be specified in the above command. To print the wave forms in xgraph format use the -x option. For complete help on using psw3 see
psw3 -h
Example:  The output file for the netlist filter.scn includes the wave form "nout-xb2.xa1.am1:w" (the noise PSD at node "nout", due to the white noise contribution from the op-amp "xb2.xa1.am1") and the wave form "nout" (the total noise PSD at node "nout" due to white and 1/f contribution of all noisy elements.

To print these two wave forms in table format use:

psw3 frequency nout-xb2.xa1.am1:w nout < sw3.noise

Note, that "frequency" is also a name of the wave form containing the frequency data.

To print the two wave forms in xgraph format use:

psw3 -x nout-xb2.xa1.am1:w nout < sw3.noise

Note, that for xgraph (two column) format, the "frequency" wave form is automatically selected as the sweep wave form, i.e. each specified wave form will be plotted with respect to frequency.
