Tutorial on how to use HSPICE

  1. First you must obtain access to a computer that has the HSPICE software installed.  Follow the steps described here.

     

  2. Now you should be able to use the remote computer almost as if you were sitting in front of it.  Preferably you should create an ordered file structure for your simulations.  You may do this by issuing the following commands:

    cd

    mkdir E3301

    cd E3301

    mkdir Example1

    cd Example1

    which will create and put you in the work directory ~/E3301/Example1

     

  3. Now download "this file" to the newly created work directory and have a look at it using your favorite editor.  I prefer the emacs editor (another piece of software that you are not likely to outgrow anytime soon), so I would issue the command:

    emacs example1.sp &

    which shortly thereafter should display a window on your local computer screen.  If this does not work, please consult one of the TAs during their office hours.  If you forget to append the "&" symbol at the end of the command, your xterm will be unavailable until emacs is terminated.  If that happens, you should close emacs and try again.

     

  4. The circuit description file, example1.sp, does not require any editing.  You may thus fire up HSPICE by issuing the following command:

    hspice example1.sp > example1.out

    Three output files should thereby be generated: example1.out, example1.tr0, example1.sw0. Each file corresponds to separate types of simulations.  The *.out file comprises diagnostic information for the simulation itself (always check this file first if you get unexpected results) as well as the result of the operating point simulation, i.e., if it is requested.
