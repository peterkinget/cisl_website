Getting Access to Computers Running HSPICE

To run HSPICE, you will need an account on the SUN workstations that are located in S.W. Mudd 12xx. 

Of logistic reasons, you will not have physical access to the SUN workstations.  Hence, to use HSPICE, you must have physical access to another networked computer with an X interface.  You may, e.g., use your CUNIX account (obtain one here) and the facilities provided in the computer lab located in S.W. Mudd room 251.  Alternatively, you may use the ILAB computers located on the 12th floor in S.W. Mudd, but for those you will need a separate account.

To use HSPICE through an X interface, you must complete the following steps:

  1. Log in on your local (physical access) computer and start a regular X terminal (xterm).
  2. Choose from this list a remote computer that has HSPICE installed.  In the following we will assume that the computer's name is name.cisl.columbia.edu
  3. Allow the remote computer to display windows on the local computer's terminal:  This is done by issuing in the xterm on the local computer the following command:

    xhost +name.cisl.columbia.edu

     

  4. Start a ssh session to the remote computer.  This is done by issuing in the xterm the following command:

    ssh login@name.cisl.columbia.edu

    and when prompted providing your password for the remote computer.

     

  5. If this is the first time you log in on any of the SUN workstations, you should change your initial password.  This is done by issuing the following command:

    passwd

    and answering the questions when prompted .

     

  6. Now you need to inform the remote computer where it should dispIay any windows through the X interface. Let local.computer.name be the computer at which you are sitting. You need to set the environment variable $DISPLAY to local.computer.name:0. How to do this depends on which shell you are running on the remote computer.  If you have a preference for not running the so-called "tcsh" shell, you probably already know how to do this in your preferred shell.  Otherwise I suggest that you use tsch.  This is done by issuing the following two commands:

    /usr/bin/tcsh

    setenv DISPLAY local.computer.name:0

     

Now you should be able to use the remote computer almost as if you were sitting in front of it.  Continue by completing the tutorial.
